Key Concepts

  • Mixamo tradeoff: fast auto-rig and library animations but limited editability without the Mixamo Rig add-on
  • Mixamo Rig add-on (zip install from GitHub) generates IK/FK controls similar to Rigify
  • Character mesh must be continuous (no floating parts) for Mixamo auto-rigging to succeed
  • FBX axis fix: in global space, character must face opposite of the green Y arrow before exporting
  • Verify axis in Mixamo upload preview: character should face toward you; if not, go back and fix orientation
  • Pose-to-pose blocking: set constant interpolation, establish key poses (wind-up, launch, peak, land, stand)
  • Bouncing ball gravity principle applies to jump: more time at peak, less time at takeoff and landing
  • Breakdown keys fill in between blocking poses; switch to Bezier then refine with graph editor handles
  • Scale keyframes in dope sheet by placing playhead at frame 0 and pressing S to retime animation

Key Concepts

  • Ball rolling: animate rotation in sync with translation; ball should visually roll along surface
  • Run cycle: hips need up-down movement in addition to rotation; locked torso reads as stiff and disconnected
  • Silhouette readability: pose camera to show line of action; avoid hiding the arc behind a straight-on view
  • Cinematography rule: place character so there is more space in the direction they are looking or moving
  • Contrast matters: character color must contrast against background; gray on dark gray hides the animation
  • Motion blur is essential for fast-moving objects; add in After Effects if not rendered in Blender
  • Cartoony vs realistic Rube Goldberg: standard version is physics-accurate; exaggerated version uses squash, stretch, smear
  • Week 12 project: two Rube Goldberg sequences (standard + cartoon exaggerated) and a jump-over-ball animation
